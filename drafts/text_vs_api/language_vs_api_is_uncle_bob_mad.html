Language vs. API (is Uncle Bob mad?)

This is the late reaction on Uncle Bob's “STOP USING SQL” blog post. The problem with these responses is that they tend to be polarizaed. Well, you have to be motivated to write a response and what motivates better than a strong disagreement with the original article or disagreement with people who strongly disagree with it. This makes public opinion seem much more radical than it is.

What I propose instead is the response that writes itself based on your personal opinion. There will be a number of hypothetical situations where you would have to prefer language over API or vice versa. The answers will then get weighted and the concusion autogenerated.

Please treat it as a mere mental experiment.

def count_all_py_shell(path = '.'):
    return os.popen('ls -R "' + path +'" | grep ".py$" | wc -l').read()

def count_all_py(path = '.'):
    return len([1 for (_, _, files) in os.walk(path) for f in files if f.endswith('.py')])
