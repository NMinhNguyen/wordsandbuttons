Approximation

The word approximation comes from Latin and means something like “bringing something close enough”. We use this word when we can set up something to use instead of something else. It is a broad term, you can technically approximate apples with oranges. But since it's usually used by boring people, it usually concers numbers, functions, points, and vector fields.

Polynomial approximation is bringing a polynomial close enough to substitute something else. It works best with a set of points, since there is a nice matrix equation that does all the work in this case. Here, try it out. You can add or remove points, and also select the polynomial degree:

Polynomial degree:

The matrix comes from the orthogonality principle. In bief, we want the squared difference between the polynomial and the points to be as little as possible, so this is a minimisation problem. But accroding to the orthogonality principle, the minimum for the sum of squared residuals is the same point where all the residuals, put in a single vector, are orthogonal to the vector of polynomial values.

There is a single solution, and there is no need to run numeric methods to find it. That makes the polynomial approximation so appealing.

You might have heard term linear regresion. It is a special case of polynomial approximation, so now you know how that works too.

Interpolation

The word interpolation also comes from Latin and it roughly translates as “smoothing things in between”. When you have a set of points, you use interpolation to find values in between these points.

With polynomial interpolations, it is about finding a polynomial that runs exactly through the points we want. Here, try it out. You can add and remove points, but you can't select a polynomial degree now since you can't have this degree less than the number of your points minus one.

The matrix is called Vandermonde matrix and it comes from the simple fact that if a polynomial runs through a point then P(xi) = yi. Just write it out for every point, put it into the matrix form and here you go.

One widely known fact, you can't have your polynomial degree less than the number of points minuns one. Another less known fact is that you can have it higher. It's very easy to see. Just imagine adding a new point to the data set, the one you don't need. The polynomial is still interpolating regarding the initial set, but you now have more control over it.

Why not have both?

This is a bonus part. One of the frequently asked questions is “can we have a polynomial that will run through some points and will approximate the rest of the data set as well?”

The answer is: yes but. We don't have a nice simple formula to do that so we have to either go numeric on that, or use a hack.

Numeric approach is not so hard. Remember that interpolating polynomial degree may be gigher than a number of points in set? We can exploit that. We can compose a Vandermonde matrix out of all interpolating points and a number of fake ones. Then we will use the fake ones as variables for the numeric least square optimizer.

When the optimizer finishes, we will have our polynomial both interpolating and approximating.

The downside of this approach is that the optimizer may finish anytime between 10 nanoseconds and 6 billion years. This depends on the number of variables and the polynomial degree.

The other approach is a hack. The thing is, in practice, we never get out polynomial exactly interpolating due to the computational error. Now if we're ok with this, why don't we argee on some tolerable error and use the orthogonal principle matrix multiplying our interpolating points more and more, untill the approximating polynomial fits them within the error?

This works because what the approximation does, it minimizes the sum or squared residuals. If you put a point twice, its residual will be twice as important to the optimizer as anyone's else. If you add it 100 times, it will attract the polynomial 100 times stronger than any other point.

The brutal simplicity of this approach is its strength. You only need to optimize until you're ok with the error, and you only have one variable to alter.
