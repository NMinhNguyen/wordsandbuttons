Interactive introduction to iterative algorithms

This is not only a tongue-twister, but an actual interactice introduction. There will be a living plot illustrating a simple iterative algorithm that solves a system of 2 linear equations.

Inherently, all the iterative algorithms consist of three steps:

  1. Start somewhere.
  2. See if you got what you were looking for. If so — great, you are done!
  3. If not, do something that brings you closer to the goal. Go to the step 2.

Our example is a linear solver. It solves a system that looks like this:

a11 x + a12 y = b1
a21 x + a22 y = b2

We want to find (x, y) for every possible as and bs.

Geometrically speaking, linear equation of two variables is just a straight line in 2D space. System of two equations is a pair of lines. And the solution is the point where they intersect. It's as simple as that!

Our algorithm rationale is as follows. The solution is a point in 2D that lies on both lines simultaneously. A point in space lies further from any point on the line than its own projection on that line. So if we project any point in space on the first line, and then on the second, and then again on the first, and so on, it will get closer and closer to the solution. The point will "travel" from projection to projection. At some point that step will get too small and we will know that we are nearly at the solution.

In terms of the steps from above it looks like this:

  1. Start at any point in 2D space.
  2. Stop when the distance point travels in one iteration is less then some predefined tolerance.
  3. Project current point on one of the lines.
